📄 生成HTML页面

根据需求生成结构良好、语义正确的HTML页面

flow_generate_html 任务流程 中等复杂度 成功率: 95%

前置条件

  • 理解用户需求的能力
  • HTML基础知识: HTML结构
  • 语义化标签知识: 语义化

执行步骤

  1. 理解需求

    分析用户描述，确定需要生成的HTML页面类型和结构

    需要知识: HTML结构

  2. 规划结构

    设计页面的DOM结构，选择合适的语义化标签

    需要知识: 语义化标签 需要知识: DOM模型

  3. 编写HTML

    根据结构设计，编写符合规范的HTML代码

    需要知识: HTML语法

  4. 验证输出

    检查HTML是否符合规范，确保标签正确闭合、属性有效

决策点

❓ 是否需要内联样式？

  • ✅ 是 → 添加内联样式流程
  • ❌ 否 → 继续下一步

❓ 是否需要JavaScript交互？

  • ✅ 是 → 添加JavaScript流程
  • ❌ 否 → 输出纯HTML

❓ 是否需要外部CSS文件？

  • ✅ 是 → 生成CSS文件流程
  • ❌ 否 → 保持内联或无样式

输出结果

主要输出
HTML文件或代码片段
可选输出
CSS文件、JavaScript文件
