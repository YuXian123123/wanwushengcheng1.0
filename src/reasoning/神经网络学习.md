# 神经元的设计意义
```
w 权重 x 输入 b 偏好 y=w*x+b
1. 原理（用你的笨脑子也能懂的比喻）
还记得我说的水果店老板吗？他只看一个特征，比如颜色红不红（x）。
他内心对这个特征的重视程度叫权重（w）。
他天生对“苹果”有点偏爱，这个偏心叫偏置（b）。
他把 颜色值 × 重视程度 + 偏心 算出一个总分。
但这总分可能很大或很小，他要把它变成 0 到 1 之间的一个“可能性”，所以用一个 sigmoid 函数挤压一下。0 代表“绝对不是苹果”，1 代表“绝对是苹果”。
数学式：
z = w * x + b
输出 y = sigmoid(z) = 1 / (1 + e^(-z))
```
比如我是宠物店的老板 我想知道我面前的动物是不是猫
x就是当前动物 w就是我对猫的重视程度 b就是我对猫的偏心程度。


用损失函数（Loss Function）。
最直白的损失就是：（你猜的 - 真实答案）的平方。
公式：L = (y - y_true)^2
y：你猜的概率
y_true：真实标签（0 或 1）
L：损失，越小越好，0 代表完美
为什么平方？
因为误差有正有负，平方后全成正的，而且会把大错罚得更重。这是本小姐最讨厌的“均方误差(MSE)”。


1. 原理：从罚单倒推，修改责任人
还是这个水果店神经元，上一轮考试它砸了：
输入 x = 0.5，参数 w = 0.2, b = -0.3
输出 y = 0.45，真实标签 y_true = 0
损失 L = 0.2025（记住，这是均方误差）
现在我们要问：L 这么大，是谁的错？是 w 的错，还是 b 的错？各自要负多少责？
这个“责任大小”就是梯度。
我们顺着计算图，从 L 一路往回求导，用链式法则，算出 dL/dw 和 dL/db。
然后让参数往梯度的反方向滚一小步——因为梯度指向的方向是让损失增大的方向，我们要让损失减小，所以要逆着走。
公式：
w_new = w_old - learning_rate × dL/dw
b_new = b_old - learning_rate × dL/db
那个 learning_rate（学习率）就是“一小步”的步长。太小就学得慢，太大就直接摔死在误差悬崖下。
