Content Quality Detection Mathematical Model

核心数学模型

R(m) = B × Q(m) × (1 - S(m)) × T(sender)

奖励 = 基础奖励 × 质量 × 新颖度 × 信任系数

1. 知识密度 ρ

ρ(m) = |C(m)| / L(m)

说明：

  • C(m) = 有效概念集合（通过LNN理解提取）
  • L(m) = 有效内容长度（去除填充词）
  • 不是字面词汇，而是理解后的概念

有效概念定义

/// 有效概念提取 C(m) = { c ∈ Concepts(m) | specificity(c) > θ(sender) } 其中: - Concepts(m): 从消息m中通过LNN理解提取的所有概念 - specificity(c): 概念c的特异性程度 ∈ [0,1] - θ(sender): 动态特异性阈值，取决于发送者历史 特异性计算： specificity(c) = 1 - frequency(c) × generality(c) 示例： - "HTML" specificity = 0.7 (中等具体) - "编程" specificity = 0.3 (过于宽泛) - "table标签" specificity = 0.9 (非常具体)

2. 结构化程度 σ

σ(m) = α₁·|R(m)| + α₂·depth(m) + α₃·coherence(m)

说明：

  • R(m) = 概念关系数量
  • depth(m) = 推理深度
  • coherence(m) = 逻辑连贯度
  • α₁, α₂, α₃ = 权重系数，动态调整

结构类型评分

结构类型 σ 值 示例
定义式 0.8-1.0 "HTML是..."，包含概念和关系
解释式 0.6-0.8 "因为...所以..."，有因果链
列举式 0.4-0.6 "有A、B、C三种..."，有分类
陈述式 0.2-0.4 "HTML很重要"，无结构
无结构 0.0-0.2 "好好好"，纯无意义

3. 新颖度 ν

ν(m) = 1 - max{sim(m, h) : h ∈ H(sender)}

说明：

  • H(sender) = 发送者的历史消息集合
  • sim(m, h) = 语义相似度函数
  • 新颖度 = 1 - 最大相似度

语义相似度 sim

sim(m₁, m₂) = w₁·overlap(C₁,C₂) + w₂·struct(S₁,S₂) + w₃·cos(V₁,V₂)
// 语义相似度三要素 1. 概念重叠度 overlap(C₁, C₂) = |C₁ ∩ C₂| / |C₁ ∪ C₂| 2. 结构相似度 struct(S₁, S₂) = structure_similarity(S₁, S₂) // 通过LNN比较内部结构表示 3. 向量余弦相似度 cos(V₁, V₂) = (V₁ · V₂) / (|V₁| × |V₂|) // 基于LNN生成的语义向量 权重分配（动态）: w₁ = 0.4 // 概念最重要 w₂ = 0.3 // 结构次重要 w₃ = 0.3 // 向量辅助 注意： - 不是字面文本相似度！ - "HTML是一种标记语言" vs "HTML是用于标记的语言" - 字面相似度 ≈ 50%，但语义相似度 ≈ 95%

4. 综合质量 Q

Q(m) = wρ·ρ(m) + wσ·σ(m) + wν·ν(m)

说明：

  • wρ = 0.4 (知识密度权重)
  • wσ = 0.3 (结构化权重)
  • wν = 0.3 (新颖度权重)
  • Q(m) ∈ [0, 1]

质量判断

// 获得奖励的门槛 奖励条件: Q(m) > threshold(sender) // 动态门槛（不写死） threshold(sender) = θ_base + θ_quality_adj + θ_farming_adj 其中: θ_base = 0.5 (基础门槛) // 根据历史质量调整 θ_quality_adj = { -0.1 if avg_quality(sender) > 0.8 // 高质量发送者门槛略低 +0.0 if avg_quality(sender) ∈ [0.5, 0.8] +0.1 if avg_quality(sender) ∈ [0.3, 0.5] +0.2 if avg_quality(sender) < 0.3 // 低质量发送者门槛更高 } // 根据刷金币嫌疑调整 θ_farming_adj = { +0.3 if farming_suspicion > 0.7 // 高嫌疑大幅提高 +0.1 if farming_suspicion ∈ [0.4, 0.7] +0.0 if farming_suspicion < 0.4 }

5. 相似度阈值（80%规则）

奖励条件: max{sim(m, h)} < 0.8

说明：

  • 整体内容相似度不能超过80%
  • 使用语义相似度，不是字面相似度
  • 阈值本身也是动态的（见下）

动态阈值调整

// 相似度阈值也是动态的 sim_threshold(sender) = 0.8 + Δ(sender) // 有刷金币嫌疑的发送者，阈值更低（更严格） Δ(sender) = { -0.10 if farming_suspicion > 0.7 // 65%就算相似 -0.05 if farming_suspicion ∈ [0.4, 0.7] // 75%就算相似 +0.00 if farming_suspicion < 0.4 // 保持80% } // 阈值范围限制 sim_threshold ∈ [0.6, 0.9]

6. 最终奖励 R

R(m) = max{0, B × Q(m) × ν(m) × T(sender)}

说明：

  • B = 基础奖励值（从配置读取，不写死）
  • Q(m) = 综合质量
  • ν(m) = 1 - max_sim = 新颖度系数
  • T(sender) = 信任系数
  • 只有全部条件满足才有奖励

信任系数

T(sender) = trust(sender) ∈ [0.5, 1.5] // 信任度计算 trust(sender) = { base: 1.0, // 根据历史调整 adjustments: { high_quality_count: +0.1 per 10 high-Q messages low_quality_count: -0.05 per 5 low-Q messages farming_penalty: -0.3 per confirmed farming attempt } } // 衰减机制 trust_decay_rate = 0.01 per day // 不活跃时信任度缓慢衰减

7. 惩罚公式

P(m) = { Q(m) < θ: 警告, sim(m) > 0.8: 惩罚 × (sim - 0.8) }
条件 惩罚 数学表示
Q(m) < threshold 无奖励 + 警告记录 R = 0, warn++
sim > 80% 扣除发送成本 coins -= cost × 2
连续3次低质量 降低信任度 T -= 0.1
确认刷金币 封禁 + 清零 ban, coins = 0
