核心问题：防刷金币

⚠️ 攻击场景分析：
  • 重复发送 - 蛊虫用相同话语刷金币
  • 微小修改 - 改几个字绕过检测
  • 合谋刷币 - 多只蛊虫互相"感谢"
  • 无意义内容 - 发送"好好好"之类废话
🎯 解决原则：
  • 质量门槛 - 不是通信就有金币，要达到质量门槛
  • 相似度检测 - 整体内容相似度不能超过80%
  • 知识结构化 - 必须分享有价值的知识
  • 动态生成 - 检测规则不写死，动态生成

语义相似度检测（关键创新）

核心：使用LNN语义理解，不是字面匹配

"HTML是一种标记语言" 和 "HTML是用于标记的语言"
字面相似度可能只有50%，但语义相似度是95%

/// 语义相似度检测器 pub struct SemanticSimilarityDetector { lnn: Arc<LNN>, history: MessageHistory, threshold: f64, // 默认0.8，但动态调整 } impl SemanticSimilarityDetector { /// 检测消息与历史的相似度 pub fn detect(&self, message: &str, sender: &GuId) -> f64 { // 1. 通过LNN理解消息（关键！） let understanding = self.lnn.understand(message); // 2. 获取发送者历史 let sender_history = self.history.get_sender_messages(sender); // 3. 计算语义相似度（不是字面） let similarities: Vec<f64> = sender_history.iter() .map(|hist_msg| { let hist_understanding = self.lnn.understand(hist_msg); self.semantic_similarity(&understanding, &hist_understanding) }) .collect(); // 4. 返回最大相似度 similarities.into_iter().fold(0.0, f64::max) } /// 语义相似度 = 概念重叠 + 结构相似 + 向量相似 fn semantic_similarity(&self, u1: &Understanding, u2: &Understanding) -> f64 { let concept_overlap = self.concept_overlap(&u1.concepts, &u2.concepts); let structure_sim = self.structure_similarity(&u1.structure, &u2.structure); let vector_sim = self.cosine_similarity(&u1.vector, &u2.vector); 0.4 * concept_overlap + 0.3 * structure_sim + 0.3 * vector_sim } }

动态门槛机制（不写死）

/// 动态门槛计算器 pub struct DynamicThresholdCalculator { base_threshold: f64, } impl DynamicThresholdCalculator { /// 门槛不写死，根据发送者行为动态调整 pub fn calculate(&self, sender: &GuId, history: &SenderHistory) -> f64 { let base = self.base_threshold; // 1. 根据历史质量调整 let quality_adj = match history.average_quality { q if q > 0.8 => -0.1, // 高质量：降低门槛 q if q > 0.5 => 0.0, // 中等：维持 q if q > 0.3 => 0.1, // 低质量：提高门槛 _ => 0.2, // 很低：大幅提高 }; // 2. 根据刷金币嫌疑调整 let farming_adj = match history.farming_suspicion { s if s > 0.7 => 0.3, // 高嫌疑：大幅提高 s if s > 0.4 => 0.1, // 中嫌疑：提高 _ => 0.0, }; // 3. 综合并限制范围 (base + quality_adj + farming_adj).clamp(0.3, 0.9) } /// 相似度阈值也是动态的 pub fn similarity_threshold(&self, history: &SenderHistory) -> f64 { let base = 0.8; let adj = match history.farming_suspicion { s if s > 0.7 => -0.1, // 高嫌疑：65%就算相似（更严格） s if s > 0.4 => -0.05, // 中嫌疑：75%就算相似 _ => 0.0, // 低嫌疑：保持80% }; (base + adj).clamp(0.6, 0.9) } }

惩罚机制

违规类型 检测条件 惩罚措施
低质量内容 Q < threshold 无奖励，记录警告
高相似度 sim > 80% 无奖励，扣除发送成本
疑似刷金币 多次高相似度 扣除额外金币，降低信任度
确认刷金币 行为模式匹配 封禁通信权限，清零金币
