置信度涌现机制

当多个验证者独立验证时，置信度通过涌现机制产生：

置信度 = (Σᵢ wᵢ × vᵢ) / Σᵢ wᵢ

其中：wᵢ = 验证者信任度 × 验证者专业度
vᵢ ∈ {0, 1} 为验证结果

签名链验证

每条知识都有完整的签名链，可追溯到原始提交：

SignChain(K) = [Sign(Gu_0, K_0), Sign(Gu_1, K_1), ..., Sign(Gu_n, K_n)]

验证条件：∀i, Verify(Sign(Gu_i, K_i)) = true

验证代价函数

验证需要消耗计算资源，代价函数优化验证策略：

Cost(验证) = α × 深度 + β × 验证者数 + γ × 知识复杂度

其中 α, β, γ 为系统参数（可配置）

置信度计算公式

单验证者置信度：
Cᵢ = 专业度ᵢ × 信任度ᵢ × 验证质量ᵢ

综合置信度：
C_total = (Σᵢ Cᵢ × vᵢ) / Σᵢ Cᵢ

入库条件：
C_total ≥ θ_共识 ∧ 一致率 ≥ θ_一致

最优验证策略

在给定预算下，最大化验证效果：

max Σᵢ (收益ᵢ - Costᵢ)
s.t. Σᵢ Costᵢ ≤ Budget

解：按 (收益/Cost) 降序选择验证任务

验证质量评估

质量指标 公式 说明
准确率 TP / (TP + FP) 正确验证比例
召回率 TP / (TP + FN) 发现问题比例
F1分数 2 × P × R / (P + R) 综合评估
