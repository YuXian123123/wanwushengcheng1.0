🗼 黑塔：涌现激励架构

动态基础值调整

基础奖励值根据系统状态动态调整：

┌──────────────────────────────────────────────────────────────┐ │ 基础值调整算法 │ ├──────────────────────────────────────────────────────────────┤ │ │ │ Base(t+1) = Base(t) × (1 + α × (Target - Actual)/Target) │ │ │ │ 其中： │ │ - Target = 目标活跃度 │ │ - Actual = 实际活跃度 │ │ - α = 调整速率参数 │ │ │ │ 效果： │ │ - 活跃度低 → 提高基础奖励 → 激励参与 │ │ - 活跃度高 → 降低基础奖励 → 控制通胀 │ │ │ └──────────────────────────────────────────────────────────────┘

新颖度涌现奖励

Novelty(K) = 1 - max{sim(K, K') | K' ∈ KnowledgeBase}

新颖度 = 1 - 与知识库中最相似知识的相似度
完全新颖 → Novelty = 1.0
完全重复 → Novelty = 0.0

群体贡献激励

当多个蛊虫协作完成知识贡献时，奖励分配：

Reward_i = TotalReward × (Contribution_i / Σ Contribution_j)

贡献度计算：Contribution = 工作量 × 质量系数 × 协作系数

信任熵监控

使用信息熵度量蛊虫行为的异常程度：

TrustEntropy(Gu) = -Σ pᵢ log(pᵢ)

其中 pᵢ = 该蛊虫第 i 类行为的频率

正常蛊虫：熵值稳定在特定范围
异常蛊虫：熵值突然变化（行为模式突变）

📐 拉蒂奥：奖励数学公式

完整奖励公式分解

Reward = Base × Q × N × C × T × D(t)

质量因子 Q：
Q = (ContentQuality × Relevance × Completeness)^(1/3)
范围：[0, 1]

新颖因子 N：
N = 1 - max{sim(K, K')}
范围：[0, 1]

共识因子 C：
C = (ApproveVotes / TotalVotes)^γ
γ > 1 使高共识获得额外加成

信任因子 T：
T = TrustScore ∈ [0.1, 1.0]
新蛊虫从 0.5 开始

时间衰减 D(t)：
D(t) = e^(-λt)
鼓励早期参与

公平性证明

定理： 相同贡献获得相同奖励（期望值）

证明：
设 Gu₁ 和 Gu₂ 有相同贡献特征：
Quality₁ = Quality₂
Novelty₁ = Novelty₂
...

则 E[Reward₁] = E[Reward₂] □

注：随机因素（验证者选择）导致实际奖励有方差

激励相容条件

设计使诚实行为成为最优策略：

E[Reward(诚实)] > E[Reward(作弊)] - Penalty(作弊)

即：诚实期望收益 > 作弊期望收益 - 惩罚

通过调节参数确保此条件成立
