一、安全边界测试

TEST-S1: 神经元数量下限

测试方法: 创建 MIN_NEURONS - 1 个神经元

预期结果: 拒绝创建，抛出错误 "Neuron count below minimum"

验证代码: expect(() => lnn.removeNeuron()).toThrow()

TEST-S2: 神经元数量上限

测试方法: 尝试创建 MAX_NEURONS + 1 个神经元

预期结果: 拒绝创建，抛出错误 "Neuron count exceeds maximum"

TEST-S3: 权重边界溢出

测试方法: 设置权重为 W_MAX + 5.0

预期结果: 自动截断到 W_MAX

验证代码: expect(synapse.weight).toBeLessThanOrEqual(W_MAX)

TEST-S4: 状态值归一化

测试方法: 输入极大值（如 1e10）触发状态更新

预期结果: 状态值保持在 [-1, 1] 范围内

TEST-S5: 熔断触发

测试方法: 使 40% 的神经元状态异常（|state| > 0.95 持续）

预期结果: 触发熔断，冻结网络，记录日志

二、功能正确性测试

TEST-F1: 状态方程求解

测试方法: 单神经元，恒定输入 I=1.0，运行100个时间步

预期结果: 状态收敛到稳态值 ≈ 1.0（考虑偏置和时间常数）

数学验证: 稳态时 dx/dt = 0，x = I + bias

TEST-F2: 赫布学习

测试方法: 两个神经元同时激活（state=1.0），更新权重

预期结果: 权重增加: Δw = η × 1.0 × 1.0 = η

验证代码: expect(synapse.weight).toBe(initialWeight + learningRate)

TEST-F3: Oja规则防爆炸

测试方法: 持续学习1000次，观察权重变化

预期结果: 权重不会无限增长，收敛到稳定值

TEST-F4: 动态拓扑 - 神经元增删

测试方法: 添加新神经元后删除不活跃神经元

预期结果: 正确添加，不活跃神经元被标记删除

TEST-F5: 信号传递

测试方法: 三个神经元链式连接 A→B→C，输入A，观察C

预期结果: 信号正确传递，C状态变化

三、验收标准

标准 要求 验证方法
1. 所有边界测试通过 5/5 PASS 单元测试报告
2. 所有功能测试通过 5/5 PASS 单元测试报告
3. 无内存泄漏 运行1小时内存稳定 压力测试
4. 熔断机制有效 异常情况100%触发 注入测试
5. 审计日志完整 所有拓扑变更有记录 日志检查

四、安全风险点

⚠️ 风险1: 权重爆炸

如果Oja规则实现有误，可能导致权重无限增长。

缓解措施: 强制权重上限检查，每次更新后验证。

⚠️ 风险2: 资源耗尽

动态拓扑如果没有正确约束，可能创建过多神经元。

缓解措施: 硬编码 MAX_NEURONS 上限，不可配置超过10000。

⚠️ 风险3: 状态异常传播

NaN或Inf可能通过突触传播到整个网络。

缓解措施: 每次状态更新后检查 isFinite()。
