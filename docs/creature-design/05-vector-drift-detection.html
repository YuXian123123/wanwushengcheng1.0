向量漂移检测与纠正

安全天才 螺丝咕姆 | Vector Drift Detection

问题描述

风险：概念向量在多次学习更新后可能发生漂移，导致：
  • 语义偏离：向量不再代表原始概念
  • 相似度失真：概念间关系变得不准确
  • 推理错误：基于漂移向量的推理结果不可靠

检测机制

pub struct VectorDriftDetector {
    // 最大允许漂移距离
    max_drift_distance: f64,
    // 历史向量快照
    vector_snapshots: HashMap<String, Vec<ConceptVector>>,
    // 漂移计数器
    drift_counters: HashMap<String, u32>,
}

impl VectorDriftDetector {
    /// 检测向量漂移
    pub fn detect_drift(&self, concept_id: &str, current: &ConceptVector) -> DriftStatus {
        // 获取原始向量
        let original = self.get_original_vector(concept_id);

        // 计算漂移距离
        let drift_distance = original.cosine_distance(current);

        if drift_distance > self.max_drift_distance {
            DriftStatus::ExcessiveDrift {
                distance: drift_distance,
                threshold: self.max_drift_distance,
            }
        } else if drift_distance > self.max_drift_distance * 0.7 {
            DriftStatus::Warning {
                distance: drift_distance,
            }
        } else {
            DriftStatus::Normal
        }
    }
}

纠正机制

策略1：锚点纠正

保留原始向量作为锚点，当漂移超过阈值时回拉

策略2：弹性约束

在学习过程中施加弹性约束，限制单次更新幅度

策略3：周期性校准

定期检查所有向量，对异常向量进行重新校准

安全边界

// 配置参数
pub struct DriftConfig {
    // 最大漂移距离：0.3（语义相似度不低于0.7）
    pub max_drift_distance: f64,
    // 单次更新最大幅度：0.05
    pub max_update_magnitude: f64,
    // 校准周期：每1000次更新
    pub calibration_interval: u32,
    // 触发纠正的漂移次数阈值：5次
    pub drift_count_threshold: u32,
}
