属性系统

蛊虫的基础属性是动态的，会随着学习、推理、交互而变化：

属性 初始值 变化机制
认知币 100 学习+推理消费+任务奖励
学习能力 1.0 学习次数 × 学习效率
推理深度 0 推理次数 × 成功率
知识广度 0 概念数量 × 覆盖率
创造力 0.1 新颖推理 × 组合创新

属性计算公式

// 学习能力增长 learning_ability = base_ability × (1 + learning_count × learning_efficiency) // 推理深度 reasoning_depth = successful_reasoning × average_confidence × depth_factor // 知识广度 knowledge_breadth = concept_count × (1 + cross_domain_connections) // 创造力（最复杂） creativity = novelty_score × combination_count × (1 + domain_span)

能力生长机制

1. 学习→能力转化

// 学习过程转化为能力 learn_to_ability(knowledge: Knowledge) → Ability { // 1. 提取知识核心概念 concepts = extract_concepts(knowledge) // 2. 分析知识结构 structure = analyze_structure(knowledge) // 3. 理解知识原理 principles = understand_principles(knowledge) // 4. 构建能力模板 template = build_ability_template(concepts, structure, principles) // 5. 内化为能力 ability = internalize(template) return ability }

2. 能力生长流程

  • 阅读阶段：阅读知识文档，提取概念和结构
  • 理解阶段：理解知识原理，建立认知模型
  • 抽象阶段：提取通用模式，形成能力模板
  • 内化阶段：将模板融入自己的能力体系
  • 验证阶段：通过实践验证能力有效性

3. 能力熟练度

能力组合与进化

组合机制

// 能力组合 combine_abilities(a1: Ability, a2: Ability) → CombinedAbility { // 检查兼容性 if !compatible(a1, a2) { return None } // 寻找组合点 intersections = find_intersections(a1.concepts, a2.concepts) // 创建组合能力 combined = CombinedAbility { base_abilities: [a1.id, a2.id], synergy: calculate_synergy(a1, a2), new_concepts: merge_concepts(a1, a2), proficiency: (a1.proficiency + a2.proficiency) / 2, } return combined }

进化路径

进化类型 触发条件 结果
熟练进化 熟练度 > 0.9 能力效率提升
组合进化 两个能力协同使用 新组合能力诞生
跨界进化 跨领域应用成功 能力适用范围扩大
创造进化 产生新颖解决方案 创造力属性提升

实现要点

  • 能力注册机制：学习完成后自动注册新能力
  • 能力索引系统：按标签、来源、熟练度索引能力
  • 能力调用接口：统一的调用接口，支持动态能力
  • 能力衰退机制：长期不用的能力熟练度会衰退
  • 能力继承机制：死亡蛊虫的能力可被其他蛊虫继承
