一、核心理念

1.1 设计哲学

核心思想：知识不是存在数据库里的一堆数据，而是无数概念簇通过互相引用形成的活的网络。每个概念都是一个微型HTML文件，既是存储单元，也是网络节点。

1.2 为什么用HTML微文件？

🔍 人类可读

HTML是纯文本，任何文本编辑器都能打开查看和编辑，方便调试和理解。

🔗 天然链接

HTML的``标签天然支持超链接，概念之间可以互相引用形成网络。

📱 轻量级

每个概念文件只有几KB，加载快速，存储高效。

🌐 可移植

HTML是通用格式，不依赖特定软件，可以在任何系统上使用。

1.3 知识图谱 vs 传统存储

特性 传统数据库 HTML知识图谱
存储形式 表结构、行记录 概念文件、引用网络
关系表达 外键、JOIN查询 超链接、直接跳转
可读性 需要工具查看 浏览器直接打开
分布性 集中式存储 可分布式、可复制
演化能力 需要迁移脚本 文件独立演化

包含关系 has-part A包含B 苹果 has-part 果核
属性关系 has-property A具有属性B 苹果 has-property 红色
因果关系 causes A导致B 下雨 causes 地面湿

前置关系 requires A需要B 写代码 requires 编程知识
顺序关系 precedes A在B之前 设计 precedes 编码

// 懒加载概念 class LazyConceptLoader { private cache: Map<string, Promise<Concept>> = new Map(); async load(id: string): Promise<Concept> { // 检查缓存 if (this.cache.has(id)) { return this.cache.get(id)!; } // 检查是否在加载中 const loading = this.cache.get(id); if (loading) { return loading; } // 开始加载 const promise = this.doLoad(id); this.cache.set(id, promise); return promise; } private async doLoad(id: string): Promise<Concept> { const concept = await loadConcept(id); // 预加载直接相关概念（一层） for (const ref of concept.relations.slice(0, 5)) { this.load(ref.targetId); // 不等待，后台加载 } return concept; } }

十、渐进式填充机制

10.1 核心问题

⚠️ 现实场景：

知识不是一次性完整的，而是一点一点补全的：

  • 蛊虫第一次听说"量子计算"时，可能只知道名字
  • 学习后，补充了基本原理
  • 再学习，补充了应用场景
  • 不断交互，知识逐渐丰富

10.2 知识完整性级别

级别 名称 包含内容 触发条件
0 空壳概念 仅ID和名称 首次遇到新概念
1 初步概念 名称 + 简短定义 + 父概念 第一次学习
2 基本概念 + 属性 + 示例 + 相关概念 多次交互学习
3 完整概念 + 向量 + 消歧义 + 详细说明 深入学习
4 专家概念 + 流程关联 + 领域知识 + 边界情况 专业领域精通

10.6 智能合并算法

// 知识合并器 class KnowledgeMerger { merge(existing: ConceptHTML, newInfo: KnowledgeUpdate): ConceptHTML { const result = existing.clone(); // 1. 只更新缺失或更详细的内容 for (const section of ['definition', 'attributes', 'examples']) { if (newInfo[section] && (result.isMissing(section) || newInfo.isMoreDetailed(section))) { result.updateSection(section, newInfo[section]); } } // 2. 关系列表：合并而非替换 if (newInfo.relations) { for (const rel of newInfo.relations) { if (!result.hasRelation(rel)) { result.addRelation(rel); } } } // 3. 向量：有新数据时更新 if (newInfo.vector && result.vectorStatus === 'uninitialized') { result.setVector(newInfo.vector); } // 4. 更新完整性级别 result.updateLevel(this.calculateLevel(result)); // 5. 记录变更历史 result.addVersionHistory({ change: newInfo.changeDescription, source: newInfo.source, timestamp: Date.now() }); return result; } calculateLevel(concept: ConceptHTML): number { let level = 0; if (concept.hasDefinition()) level++; if (concept.hasAttributes()) level++; if (concept.hasExamples()) level++; if (concept.hasVector()) level++; if (concept.hasDomainKnowledge()) level++; return Math.min(level, 4); } }

10.7 缺失知识查找

// 查找缺失知识 class KnowledgeGapFinder { // 查找所有不完整的概念 async findIncomplete(): Promise<ConceptRef[]> { const concepts = await this.scanAllConcepts(); return concepts.filter(c => c.level < 3); } // 查找特定概念的缺失部分 getGaps(conceptId: string): KnowledgeGap[] { const concept = this.load(conceptId); const gaps: KnowledgeGap[] = []; if (concept.isMissing('definition')) { gaps.push({ field: 'definition', priority: 'high', suggestion: '需要基本定义' }); } if (concept.isMissing('examples')) { gaps.push({ field: 'examples', priority: 'medium', suggestion: '收集使用示例' }); } if (concept.vectorStatus === 'uninitialized') { gaps.push({ field: 'vector', priority: 'high', suggestion: '需要训练概念向量' }); } return gaps; } // 生成学习计划 generateLearningPlan(creature: Creature): LearningTask[] { const incomplete = this.findIncomplete(); const gaps = incomplete.flatMap(c => this.getGaps(c.id)); // 按优先级和与蛊虫的相关性排序 return gaps .sort((a, b) => this.priorityScore(b) - this.priorityScore(a)) .slice(0, 10); // 每次最多10个学习任务 } }
