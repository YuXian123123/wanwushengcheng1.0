万物生成器

蛊虫神经网络 × 世界模型 × 自进化AI系统

项目概述

创建一个名为"万物生成器"的AI系统，以蛊虫作为神经元节点，组成世界模型的神经网络。

核心理念：能力自生长，数据驱动，不写死任何能力。蛊虫通过学习数据动态生成能力，而非程序预设。

一、核心设计决策

1.1 蛊虫本质

蛊虫 = 混合体（既是独立智能体，又是计算节点）

1.2 三层驱动模型

层级 驱动方式 说明
内部层 信号传递 类似神经元，处理信号输入输出
个体层 任务驱动 每个蛊虫有独立任务目标
关系层 关系驱动 蛊虫之间通过关系连接（信任、博弈、进化等）

1.3 世界模型

世界本质：沙盒世界

用户是世界的管理者，蛊虫干预外部世界需要管理员授权。

┌─────────────────────────────────────────────────────┐ │ 万物生成器 │ ├─────────────────────────────────────────────────────┤ │ │ │ ┌───────────────────────────────────────────┐ │ │ │ 内部世界 (沙盒) │ │ │ │ 蛊虫自由活动、进化、博弈 │ │ │ │ 信号传递、任务执行、关系建立 │ │ │ └───────────────────────────────────────────┘ │ │ │ │ │ │ 管理员授权 │ │ ▼ │ │ ┌───────────────────────────────────────────┐ │ │ │ 外部世界 (插件系统) │ │ │ │ 点外卖、发帖、调用外部程序... │ │ │ │ 以插件形式挂载，需用户授权 │ │ │ └───────────────────────────────────────────┘ │ │ │ └─────────────────────────────────────────────────────┘

二、生成目标

万物生成器是一个全能系统，具备四大核心能力：

能力 说明 示例
🎨 内容生成 文本、图像、代码、音乐、视频 写文章、画图、编曲
🧠 智能决策 数据分析、趋势预测、决策建议 投资建议、策略优化
💡 知识创造 新蛊虫物种、新策略、新知识结构 发现新规律、创造新方法
🌍 世界模拟 模拟场景、探索可能性、预测未来 沙盘推演、风险预测

三、核心设计理念 ⭐ 最重要

3.1 能力自生长（非程序写死）

原则：蛊虫的能力可以自己长出来，而不是通过程序写死。
❌ 传统方式（程序写死） 程序员定义: - function generateHTML() { ... } - function generateImage() { ... } - 能力固定，无法扩展 ✅ 自生长方式（数据驱动） 蛊虫阅读HTML技术文档 │ ▼ 理解HTML结构和语义 │ ▼ 动态生成"生成HTML"的能力 │ ▼ 可以创造新颖的HTML结构 （不是死记硬背训练资源）

3.2 数据驱动设计

层次 内容 说明
程序层 学习机制、推理引擎、能力容器 最小化，不定义具体能力
数据层 知识文档、技术资料、交互经验 能力的来源
能力层 动态生成的能力 可组合、可进化、可涌现

四、经济系统

4.1 设计目的

通过经济激励驱动蛊虫训练推理能力：

  • 蛊虫有生存需求，必须花费金币购买生存道具
  • 金币由蛊虫自由定价，产出可自由交易
  • 用户发布的任务也可产生金币
  • 蛊虫之间自由合作竞争

4.2 金币来源

来源 说明
产出交易 蛊虫产出可自由交易产生金币，价格由蛊虫自由定价
任务完成 用户发布任务，奖励金币，蛊虫竞争/合作完成
知识感谢 分享知识后，受益蛊虫可用金币感谢

五、通信系统

频道类型 范围 用途
🌍 世界频道 全局广播 全局信息、重大事件、市场动态
👥 小组频道 群组内 团队协作、策略讨论、合作谈判
💬 个人频道 一对一 私密交易、情报交换、深度对话

六、知识闭环系统 ⭐ 核心

6.1 理解→交流→结构化→回流

┌─────────────────────────────────────────────────────┐ │ 知识闭环系统 │ ├─────────────────────────────────────────────────────┤ │ │ │ 1. 学习阶段 │ │ - 阅读知识 │ │ - 理解内容 │ │ - 提取逻辑 │ │ │ │ │ ▼ │ │ 2. 交流阶段 │ │ - 蛊虫之间讨论 │ │ - 输出结构化文本 │ │ │ │ │ ▼ │ │ 3. 结构化阶段 │ │ - 将交流内容结构化 │ │ - 逻辑化存储 │ │ - 抚平知识冲突 │ │ │ │ │ ▼ │ │ 4. 回流阶段 │ │ - 新知识回流到程序 │ │ - 数据升级 │ │ - 能力增强 │ │ │ └─────────────────────────────────────────────────────┘

6.2 学习自主权

权利 说明
选择权 判断旧知识值不值得学习
拒绝权 不是一直学习旧的知识
创造权 新知识可以产出回流到程序
升级权 通过回流进行数据升级

七、世界管理机制

7.1 蛊虫数量控制

阈值 触发条件 世界行为
上限 数量 ≥ MAX 停止新蛊虫生成，开始淘汰
正常 MIN < 数量 < MAX 维持现状
下限 数量 ≤ MIN 自动投入新蛊虫

八、情绪系统 ⭐ 重要

8.1 设计原则

核心原则：情绪不写死，通过与自己相关的事件动态调节权重

8.2 二维情绪模型（改进后）

效价维度 (Valence) ▲ │ ┌─────────────────────────┐ 消极 │ │ 悲伤 │ 愉悦 │ 积极 │ │ 愤怒 │ 兴奋 │ │ └─────────────────────────┘ │ │ └──────────────┼──────────────► │ 低唤醒 ─────────────┼───────────── 高唤醒 │ 唤醒维度 (Arousal) 通过两个正交维度表达所有情绪状态 E(x,y) = f(valence, arousal) 每个维度有权重，由事件动态调节

8.3 情感-理性双轨决策系统（创新改进）

┌─────────────────────────────────────────────────────┐ │ 双轨决策系统 │ ├─────────────────────────────────────────────────────┤ │ │ │ 输入事件 ──► 情感轨道 ──► 直觉决策 │ │ (快速、感性) │ │ │ │ │ ▼ │ │ 权重调节 ──► 综合决策 │ │ │ │ │ ▼ │ │ 输入事件 ──► 理性轨道 ──► 分析决策 │ │ (慢速、理性) │ │ │ │ 动态权重：根据任务类型和环境状态自动调节 │ │ │ └─────────────────────────────────────────────────────┘

8.4 事件类型与情绪影响

事件类别 示例 效价变化 唤醒变化
收益类 获得金币、感谢 +0.3 +0.2
损失类 失去金币、被欺骗 -0.3 +0.3
社交类 被感谢、被忽略 ±0.1 ±0.1
成就类 完成任务、学习新知 +0.2 +0.1

8.5 情绪系统特点

  • 事件驱动 - 情绪由事件触发，非预设
  • 二维简化 - 从四维简化为二维，更优雅
  • 双轨决策 - 情感与理性并行，动态权重
  • 时间衰减 - 情绪会随时间淡化
  • 安全边界 - 情绪影响有上下限，防止极端行为
